Ontology 建模在
Data Agent 领域的深入调研

Ontology Modeling in Data Agents: Academic Research, Open Source Ecosystem & Commercial Landscape
由 Kira Chen 整理 · 2026年4月
5 个核心章节 · 基于 Grok 深度调研
引用 700+ 来源

Ontology建模（本体论建模）在 Data Agent（数据分析 Agent、Text-to-SQL Agent、Agentic Analytics）领域已从"辅助工具"演变为核心基础设施。它通过形式化业务实体、关系、度量和规则，构建可解释、可推理的 Semantic Layer（语义层），直接解决 Agent 在 Schema Linking、幻觉、跨域一致性和自主规划上的痛点。


2025–2026 年，学术界聚焦 Agent 驱动的 Ontology Matching 与动态构建；开源项目强调 YAML 声明式 + API-first 设计；商业产品则实现平台原生集成，支持 L3/L4 自主化 Agent。Gartner 明确将"universal semantic layers"列为 2030 年前的关键基础设施（与数据平台、网络安全同级），业界共识高度一致：无 Ontology 建模的 Data Agent 难以落地生产。

01 / 学术研究

三大研究方向（2025–2026）

Ontology 在 Data Agent 中的作用主要体现在三个方向：自动化 Ontology Matching、动态 Semantic Layer 构建、Neurosymbolic 约束推理，直接服务于 Agent 流水线中的 Intent Parsing、Schema Linking 和 Planner 模块。

方向一：Agent 驱动的 Ontology Matching

Agent-OM: Leveraging LLM Agents for Ontology Matching
VLDB 2025/2026

首个纯 LLM-Agent 驱动的 Ontology Matching 框架。采用 Siamese 双代理结构（Retrieval Agent + Matching Agent）+ 共享 Memory + 专用 OM 工具集，通过 CoT 规划、RAG 和混合数据库（关系 + 向量）实现异构 Ontology 的自动对齐。在 OAEI 基准上复杂任务显著超越传统方法。开源实现已发布。

自动化构建企业级 Semantic Layer，支持多源 Schema 到 KG 的映射，解决 Schema Linking 核心瓶颈
Ontology Matching Multi-Agent 开源 OAEI 基准
A Multi-Agent System for Semantic Mapping of Relational Databases to Knowledge Graphs
arXiv 2511.06455 · 2025.11

三代理系统（Mapping Agent + Relation Agent + Validator Agent），使用 GPT-4o-mini 将关系表/列映射到 Schema.org Ontology，构建统一 KG Semantic Layer。Spider 数据集 5 个领域平均映射准确率 >90%（高置信度任务达 100%）。

为 Text-to-SQL Agent 提供 Ontology 驱动的语义 grounding，直接可落地为 Schema Linking 模块
DB→KG 映射 Schema.org Spider 基准 >90%
GenOM: LLM-Enhanced Ontology Alignment
arXiv 2508.10703 · 2025.8

LLM 生成描述 + 嵌入相似度 + 精确字符串匹配的三阶段流水线，提升 Ontology Alignment 精度。针对同义词、缩写、跨语言概念映射效果显著。

Ontology Alignment 精度提升，适用于企业多系统 Schema 统一
Ontology Alignment 嵌入+精确匹配

方向二：动态 Semantic Layer 构建

SciDataCopilot: An Agentic Data Preparation Framework
arXiv 2602.09132 · 2026.2

科学数据 Agent 框架，使用 Data Lake + Tool Lake + Case Lake 科学 Ontology 驱动 4 代理协作（Data Access / Intent Parsing / Data Processing / Data Integration）。实现异构实验数据的自主准备与 AI-Ready 转换。

Ontology 作为知识库核心驱动多代理协作，扩展到领域特定（科学、医疗）DA-Agent
科学数据 4-Agent 协作 领域 Ontology
LETHE-Chat: Schema-Aware LLM Agent for Clinical Text-to-SQL
2025/2026

结合临床医疗 Ontology（SNOMED、FHIR Schema）实现领域特定 grounding 的 Text-to-SQL Agent。Schema-Aware 机制将 Ontology 注入 Intent Parsing 阶段，显著减少医疗术语幻觉。

医疗领域 Ontology + Text-to-SQL 结合的典型案例，验证了 ontology grounding 的垂直场景价值
医疗 Ontology SNOMED/FHIR Text-to-SQL

方向三：Neurosymbolic 约束推理

Ontology-Constrained Neural Reasoning in Enterprise Agentic Systems
arXiv 2604.00555 · 2026.4

提出三层 Ontology 架构（Role Ontology 定义代理职责、Domain Ontology 定义领域知识结构、Interaction Ontology 规范交互流程）+ Neurosymbolic 架构，在 Foundation AgenticOS 平台上约束 LLM 代理的输入/输出。600+ 次实验（p<0.001）在金融、保险、医疗等企业场景中显著降低幻觉、提升合规性和角色一致性。

直接解决调研中提到的"规划一致性"和"domain drift"痛点，是 Compound AI 流水线的形式化约束层
三层 Ontology Neurosymbolic 金融/保险/医疗 600+ 实验验证
💡

总体趋势：Ontology 从静态知识库升级为 Agent 可操作、可动态构建的上下文层，与 Compound AI 流水线高度融合。学术侧重自动化对齐与神经符号融合，工业侧重治理与 grounding。

02 / 开源生态

开源项目全景

2026 年开源生态聚焦声明式 YAML 建模 + Agent API + 跨平台 SQL 编译，便于 Data Agent 直接消费。主要分为两类：Semantic Layer 框架（面向 Agent 查询接口）和 Ontology 工具（面向自动化映射与对齐）。

项目 核心特性 对 Data Agent 的价值 链接
WrenAI 带 Semantic Layer（MDL）的开源 Text-to-SQL + Text-to-Chart GenBI Agent，YAML 定义业务语义 业务定义一次编码，Agent 查询自动 grounding，避免 raw SQL 幻觉 Canner/WrenAI
OrionBelt API-first 语义引擎，YAML 模型编译为多方言 SQL（BigQuery/Snowflake/Databricks 等），内置 MCP 协议支持 Agent 专用 Query Planner，支持 MCP 协议直接集成 GitHub
dbt MetricFlow 指标层引擎（Apache-2.0 开源），YAML 定义 Metric/Dimension/Entity，跨平台 SQL 编译 Agent 可直接查询治理后的业务概念，dbt Labs 明确定位"AI Agent 与结构化数据的桥梁" dbt-labs/metricflow
Cube.dev 成熟开源 Semantic Layer（代码优先，REST/GraphQL/SQL API），多平台适配 独立于数据仓库的通用语义层，适合多源 AI 集成 cube-js/cube
Semantica 上下文图 + Semantic Layer + 决策智能框架（可解释 + 溯源 + Accountability Layer） 为 Agent 构建可审计的决策链，支持合规场景 GitHub
Agent-OM LLM-Agent Ontology Matching 工具集（Siamese 双代理，OAEI 基准验证） 自动化 Schema→Ontology 映射，动态构建企业 Semantic Layer GitHub
AWS Semantic Layer Sample 使用 Bedrock Agent 构建数据无关 Semantic Layer（Schema Discovery + Graph Traversal），完整参考架构 完全 Agent-native 参考架构，可直接 fork 验证 aws-samples
📋

Open Semantic Interchange (OSI) 规范（2026.1，Apache-2.0）：由 Snowflake / dbt / Atlan 等 30+ 组织联合维护，实现 Semantic Metadata 跨平台交换标准。这是语义层互操作性的重要基础设施，未来将显著降低 Multi-Cloud Data Agent 的集成成本。

03 / 商业产品

商业化产品格局

主流云厂商与专用厂商均已将 Ontology/Semantic Layer 作为 Agent-first 特性 GA，2026 年形成"平台原生 vs. 专用 Ontology"两极格局。

Databricks
2026.4 GA

Unity Catalog Business Semantics / Metric Views。在 Unity Catalog 中集中定义治理的 Metric Views（业务 KPI、维度、度量、过滤规则、同义词），YAML/SQL 声明式定义。Mosaic AI Agent Framework 直接消费，继承完整 RBAC/Lineage/Masking 治理。

强项：治理统一，Agent 直接消费，避免"Lost in the Middle"
Snowflake
2026.3 GA

Agent Context Layer = Semantic Model（Metric/Dimension）+ Ontology（关系/身份层）双层架构。Cortex Analyst + Semantic Views Autopilot（AI 自动建模）。Open Semantic Interchange 推动可移植性。

强项：Agent Context Layer 直接 grounding 多代理推理，AI 自动建模降低维护成本
Microsoft Fabric IQ
Preview GA

原生 Ontology Item（Entity Type + Properties + Relationships + 约束），绑定 OneLake/实时数据。Fabric Data Agent 可直接选择 Ontology 作为数据源，支持跨域推理与 Operations Agents。形式化程度最高（OWL 级）。

强项：Ontology 作为一等公民，形式化推理能力最强，专为 Data Agent 设计
Timbr.ai
Commercial

SQL Ontology 平台，Ontology-based Semantic Layer（SQL-native KG），零拷贝虚拟化，原生支持 Databricks/Snowflake/Fabric。内置 GraphRAG + NL2SQL 引擎，AI Agent 可通过 SQL 端点直接消费完整 Ontology。

强项：SQL-native 设计，无迁移成本，跨平台一致性最佳
Palantir Foundry
Commercial

企业级 Ontology-Driven 平台（数字孪生 + 动态治理）。2025 年与 Databricks 战略合作，Palantir Ontology System 与 Lakehouse 深度集成。不止分析，还支持运营执行，适合高自主化 Agent。

强项：支持 L4 自主化 Agent，运营执行能力业界最强
其他生态
多厂商

Atlan：治理 + Ontology，数据资产目录与 Agent grounding 结合。
PuppyGraph：Graph 作为 Ontology 加速 Text-to-SQL，零 ETL。
Neo4j：GraphRAG + Databricks Agent Bricks，多跳关系推理。

强项：各有垂直场景优势，可作为平台原生方案的补充层

05 / 结论

结论与落地建议

"2026 年，Ontology 建模已达成业界共识——它是 Data Agent 从 PoC 走向生产、从 Text-to-SQL 走向 L4 自主化的非谈判基础。"
01
学术层：Agent-OM / Neurosymbolic Reasoning 提供自动化构建与形式化约束方法，是 Semantic Layer 的技术底座。
02
开源层：WrenAI / dbt MetricFlow / Cube.dev 降低工程门槛，YAML 声明式 + API-first 设计让 Data Agent 直接消费语义。
03
商业层：Databricks/Snowflake/Fabric 提供平台原生集成，治理统一、无额外集成开销，是企业快速起步的首选。
04
推荐混合架构：平台原生 Semantic Layer（Databricks Metric Views 或 Snowflake Cortex）+ 专用 Ontology 工具（Timbr 或 Neo4j GraphRAG）+ 开源 dbt MetricFlow，覆盖治理、推理、成本三维需求。
05
未来方向：Semantic Layer（度量治理）与 Ontology（关系推理）融合趋势明确；OSI 标准将推动可移植性；Agent Context Layer 正成为 L3/L4 自主化 Agent 的标配基础设施。
